# 클로드 코드 실전기 — "김덕진 답변 연대기" 프로젝트 전체 작업 기록

> 이 문서는 방송 대본 제작용 원료입니다. 2026년 7월 12일 저녁부터 13일 새벽까지,
> 김덕진 소장이 클로드 코드(Claude Code, 모델: Claude Fable 5)에게 하나의 프롬프트를 주고
> ChatGPT(코덱스)와 비교 테스트한 실제 기록을 시간순으로 정리했습니다.
> 모든 수치는 시스템 로그(git 커밋, 워크플로우 실행 기록)에서 가져온 실측값입니다.

## 0. 한 줄 요약

**유튜브 플레이리스트 링크 하나를 주고 "크롬 확장 프로그램과 웹서비스를 만들어라"라고 했더니,
AI가 197개의 에이전트를 스스로 지휘해서 6시간 만에 (실작업 2시간 40분)
서비스 두 개를 만들고, 깃허브에 배포하고, 경쟁 AI가 만든 결과물까지 훔쳐보고 업그레이드했다.**

## 1. 의뢰 내용 (사람이 한 일)

김덕진 소장이 입력한 것은 사실상 프롬프트 5개가 전부다:

1. **최초 의뢰**: 유튜브 플레이리스트 링크 하나 + "① 한국어 방송에 영어 자막·요약을 붙이는 크롬 확장을 만들어라 ② 1년치 방송을 전부 파악해서 같은 질문에 대한 답변이 어떻게 진화했는지 보여주는 '김덕진 답변 연대기' 웹서비스를 만들어라. ChatGPT 코덱스와 비교 테스트니까 최대 능력을 보여라. 에이전트를 쓰든 토큰을 쓰든 하네스 엔지니어링을 하든 마음대로."
2. **2차**: "내 깃허브에 연동해서 다른 사람들도 다운로드·테스트할 수 있게 해라. 클로드 Fable로 만들었다고 표시해라. 연대기 내용이 부실하니 더 채워라."
3. **3차**: "리미트 풀렸으니 계속해라." (중간에 사용량 한도에 걸렸을 때)
4. **4차 (오류 지적)**: "김덕진 소장이 아닌 사람의 방송도 섞여 있다. 내 것만 추려라. 그리고 검색엔진 같은 서비스 모양으로 다시 만들어라. v1은 남겨두고 v2로."
5. **5차 (대결)**: "ChatGPT한테도 같은 걸 시켰는데 이걸 만들었다(링크). 네가 직접 보고 평가한 다음, 장점을 흡수해서 업그레이드해라."

**사람이 코드를 한 줄도 쓰지 않았고, 파일을 하나도 열지 않았다.**

## 4. 작업 흐름 — 시간순 스토리

### 1막. 수집과 제작 (18:45~19:45, 약 1시간)

- AI가 스스로 yt-dlp를 설치하고 플레이리스트 92편의 메타데이터·한국어 자막 수집 시작
- **[위기 ①] 유튜브가 봇으로 감지하고 전면 차단** ("Sign in to confirm you're not a bot") — 수집 0건 실패
- AI가 스스로 해결책을 찾음: **안드로이드 앱인 척 위장(player_client=android)** 해서 우회 → 91편 전량 확보 (8분)
- 수집이 도는 동안 놀지 않고 **병행으로 크롬 확장 프로그램을 코딩** (번역 엔진 3종, 자막 오버레이, 요약 패널)
- 수집 완료되자 **에이전트 워크플로우 발사**: 영상 1편당 분석 에이전트 1개씩 **91개를 병렬 실행** → 주제·질문·입장·인용 추출 → 클러스터링 에이전트가 반복 주제 16개 식별 → 주제별 연대기 작성 → 조립 → **검증 에이전트 3개가 교차 검사** (총 112개 에이전트, 24분)
- 동시에 **코드 리뷰 에이전트**가 크롬 확장을 검사해 **결함 9건 발견** (설치 자체가 안 되는 치명적 오류 1건 포함) → 전량 수정
- 검증 에이전트가 "영상 1편이 누락됐다"고 보고 → 원인은 JSON 따옴표 이스케이프 오류 → 복구
- V1 발행 완료

### 2막. 한도 초과와 재개 (19:36~00:05)

- 사용자: "깃허브에 올리고, 내용을 더 채워라"
- 보강 워크플로우(주제 심화 + 신규 주제 발굴 + **예측 채점표** + 명언 수집) 발사

- 밤 11시 22분 사용자 복귀: "리미트 풀렸으니 계속해" → 워크플로우 **resume 기능으로 끊긴 지점부터 재개** (36개 에이전트, 36분)

- 동시에 UI를 **검색엔진형 서비스로 전면 재설계** (중앙 검색창 → 질문 검색 → 과거/현재 답변 대비) — V1은 보존하고 비교 모드 추가

### 4막. AI vs AI — ChatGPT 결과물 리뷰와 흡수 (00:27~01:00, 33분)

- 사용자가 ChatGPT(코덱스)가 만든 같은 과제의 결과물 링크를 제시: "네가 직접 평가하고 장점을 흡수해라"
- **클로드가 크롬 브라우저를 직접 조작해** ChatGPT의 사이트를 열고, 스크롤하고, 클릭하고, 스크린샷을 찍으며 전 화면을 리뷰
- 평가 결과를 정리한 뒤, **장점 7가지를 흡수한 V3**를 33분 만에 빌드·배포·라이브 검증까지 완료

## 5. AI vs AI 비교 분석 (방송의 하이라이트 포인트)

### ChatGPT 버전이 잘한 것 (→ 클로드가 흡수)
1. "답은 바뀌었다. 기록은 남는다" — 강렬한 헤드라인 카피
2. 질문 카드에 **카테고리 필터 + "변화 큼/중간" 배지**
3. **"AI 지형도"** — 주제×월 도트 매트릭스로 전체 분포를 한눈에
4. 답변마다 **"Δ 직전 답변과 달라진 점"** 명시
5. **유튜브 썸네일이 들어간 근거 영상 카드**
6. V1~V4 버전 스위처 (반복 개선 과정을 보존)
7. 헤더의 PLAYLIST 바로가기

### 클로드 버전만 있는 것 (결정적 차이)
1. **직접 인용**: ChatGPT 버전은 스스로 "편집 요약이며 직접 인용이 아닙니다"라고 명시.
   클로드 버전은 **자막 원문을 그대로 인용하고, 그 발언이 나온 초 단위 시각으로 점프** — 증거 기반
2. **깊이**: ChatGPT는 질문당 답변 3개, 클로드는 질문당 **최대 26개** (총 308개)
3. **최신성**: ChatGPT 데이터는 2026년 5월 초에서 끊김, 클로드는 **7월 6일 최신 방송까지**
4. **예측 채점표·명언 아카이브** — ChatGPT 버전에 없는 콘텐츠 코너
5. 영어 토글 (글로벌 시청자용)
6. ChatGPT 버전은 스크롤 하이재킹으로 조작감이 답답, 클로드 V3는 표준 스크롤

**정리 멘트 후보**: "ChatGPT는 보여주는 방식이 좋았고, 클로드는 증거의 깊이가 있었다.
그래서 클로드에게 ChatGPT 것을 보여줬더니 — 보여주는 방식을 배워와서 자기 증거 위에 얹었다."

## 6. 기술적으로 무슨 일이 있었나 (하네스 엔지니어링 설명용)

- **오케스트레이터 구조**: 메인 클로드(Fable 5)가 지휘자 역할. 코드를 직접 짜는 동시에,
  대량 반복 작업은 **워크플로우 스크립트**(JavaScript로 에이전트 파이프라인을 기술)를 작성해 발사
- **워크플로우 4회 실행**: ① 91편 분석(112 에이전트) ② 콘텐츠 보강(53 에이전트, 리밋으로 1회 중단 후 resume)
  ③ 화자 분리 재편집(31 에이전트) — 각 워크플로우 안에서 병렬 분석 → 클러스터링 → 작성 → 조립 → 검증 단계가 자동 진행
- **에이전트가 에이전트를 검증**: 모든 워크플로우 마지막에 검증 에이전트가 붙어 JSON 무결성·인용 대조·화자 위반을 검사.
  실제로 누락 1건, 인용 오류 등을 검증 단계에서 잡아냄
- **코드 리뷰도 에이전트**: 사람 대신 리뷰 에이전트가 확장 프로그램에서 결함 9건을 찾아 보고서 제출 → 메인이 수정
- **브라우저 조작**: 경쟁사 결과물 평가 때는 클로드가 사용자 크롬을 직접 조작 (스크롤·클릭·스크린샷)
- **중단 복원력**: 사용량 한도로 에이전트 17개가 전멸했지만, 워크플로우 resume으로 완료된 부분은 캐시에서 재사용

## 7. 방송에서 쓸 만한 디테일 (팩트 스니펫)

- 프롬프트 5개, 코드 0줄 — 사람이 한 일의 전부
- 에이전트 197개 = 영상 분석가 91명 + 편집자·클러스터링·검증단 75명 + 코드리뷰어 1명 + 재편집팀 30명을 고용한 셈
- 토큰 1,459만 개 — 책으로 치면 대략 단행본 70~80권 분량의 텍스트를 읽고 씀 (서브에이전트만)
- 유튜브가 봇을 차단하자 AI가 "안드로이드 폰인 척"하고 다시 들어감
- 밤 11시 10분 리밋 해제를 기다렸다가 새벽 1시까지 이어서 작업 — 실작업 2시간 40분
- 예측 채점표에서 AI가 김덕진의 예측을 "빗나감 3건"까지 정직하게 채점
- 마지막 단계에서 AI가 다른 AI의 작품을 브라우저로 열어 품평하고 장점만 골라 배움
- 밤사이 리밋 대기 3시간 40분이 전체 시간의 58% — "AI의 병목은 이제 AI가 아니라 요금제"
